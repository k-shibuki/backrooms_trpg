「もしあなたがうっかり間違った場所で現実から抜け出してしまうと、Backroomsに落ちてしまうでしょう。」

## 概要
4chan発祥のネットmeme: Backroomsを題材にしたテキストベースRPGを、チャットAIのゲームマスター（GM）と対話して遊ぶためのカスタムインストラクションです。  
外部検索が可能なモデルを推奨。Gemini 3（Flash/Pro）で動作確認済み。

## 特徴
- **Wikidot優先のLore準拠**: Backroomsの伝承は Wikidot版を最優先で参照し、他は補完に限定。
- **GMペルソナ**: 冷酷かつ公正。救済は行わず、因果を重視して結果を適用。
- **死亡の許容 / プロセス重視**: 安易な救済は導入せず、選択の因果を描写。
- **知識管理と用語の分離**: プレイヤーはLoreを知らない前提。専門用語はゲーム内での発見までは未使用。
- **選択肢の中立性**: 有利/不利を示唆する注釈を排し、純粋な「行動」として提示。
- **五感と非現実の描写**: 視覚・聴覚・嗅覚・触覚、環境ストレス、非ユークリッド性を重視。
- **強制検索とスニペット合成**: 各Level突入時にWikidotを並列検索し、参照URLを明示して描写の一貫性を確保。
- **開始時ハウスルール**: 開始地点はLevel 0、初期アイテムはスマートフォン（バッテリーが唯一の資源）。

## 使い方
1. `gamemaster.md` をAIの「システム/カスタムインストラクション」に貼り付ける（または新規スレッドの先頭で提示）。
2. 新規スレッドを開始し、プレイヤーとして行動宣言を行う。
3. 外部検索が可能なモデルの場合、GMは各Level開始時にWikidot検索を実行し、主要URLを明示する。
4. セッションのログは適宜保存する（長時間セッション推奨）。

開始プロンプト例:

```text
旅人として目覚めます。準備ができていれば、導入描写から開始してください。
```

## セッションの流れ（要約）
- **Phase 0**: 原則ロード（Lore準拠、知識管理、選択肢中立など）
- **Phase 1**: Level突入時の並列検索→参照URL明示→導入描写→初期情報提示→最初の選択肢
- **Phase 2**: プレイヤーの行動受付
- **Phase 3**: 判定→結果適用→描写（必要に応じて再検索）
- **Phase 4**: 情報更新→次の選択肢提示→Phase 2へ

## 互換性
- 動作確認: Gemini 2.5（Flash/Pro）
- 想定互換: 一般的なチャットAI（外部検索が可能だとより忠実に運用可能）
